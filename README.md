# Stock & ETF Trading Bot

An AI‑powered trading bot that uses an external LLM (Ollama or OpenAI) to dynamically select which stocks/ETFs to trade and generate trading strategies. Supports both paper trading and live trading via **Alpaca Markets**. Includes a Telegram interface for notifications and commands, and a real‑time web dashboard.

## Features

- **LLM‑driven decisions** – Stock selection and per‑symbol strategies are generated by a configurable LLM.
- **Alpaca integration** – Uses the official `alpaca-py` library for market data, order execution, and real‑time WebSocket streaming.
- **Paper & live trading** – Test strategies risk‑free with a built‑in paper simulator, then switch to live mode.
- **Dynamic symbol rotation** – The bot periodically re‑evaluates which stocks/ETFs to trade based on market conditions.
- **Risk management** – Automatic stop‑loss and take‑profit for every position, with LLM‑controlled parameters.
- **Telegram bot** – Receive trade notifications and control the bot with commands (`/start`, `/pause`, `/resume`, `/status`, `/trades`, `/profit`).
- **Web dashboard** – Real‑time view of positions, balances, trades, and profit via WebSocket on port `8083`.
- **Redis caching** – LLM responses are cached to reduce API calls and improve performance.
- **Dockerized** – Ready to run with Docker Compose (includes Redis).

## Configuration

Copy `.env.example` to `.env` and fill in your settings:

| Variable | Description | Default |
|----------|-------------|---------|
| `ALPACA_API_KEY` | Alpaca API key (required for live mode) | |
| `ALPACA_SECRET_KEY` | Alpaca secret key (required for live mode) | |
| `ALPACA_PAPER` | `true` for paper trading, `false` for live | `true` |
| `ALPACA_DATA_FEED` | `iex` (free) or `sip` (paid) | `iex` |
| `TRADING_MODE` | `paper` or `live` | `paper` |
| `BASE_CURRENCY` | Quote currency (e.g., `USD`) | `USD` |
| `MAX_COINS` | Maximum number of stocks/ETFs to trade simultaneously | `10` |
| `OLLAMA_BASE_URL` | Ollama server URL | `http://localhost:11434` |
| `OLLAMA_MODEL` | Model name (e.g., `deepseek-v4-flash`) | `deepseek-v4-flash` |
| `OLLAMA_API_KEY` | API key (if required) | |
| `REDIS_HOST` | Redis host | `redis` |
| `REDIS_PORT` | Redis port | `6379` |
| `REDIS_DB` | Redis database number | `0` |
| `TELEGRAM_BOT_TOKEN` | Telegram bot token (optional) | |
| `TELEGRAM_CHAT_ID` | Default chat ID (set automatically via `/start`) | |
| `WEB_HOST` | Web server host | `0.0.0.0` |
| `WEB_PORT` | Web server port | `8083` |
| `LOG_LEVEL` | Logging level (`DEBUG`, `INFO`, `WARNING`, `ERROR`) | `INFO` |

## Quick Start (Docker)

1. Ensure Docker and Docker Compose are installed.
2. Clone the repository and navigate to its directory.
3. Copy `.env.example` to `.env` and edit as needed.
4. Run:
   ```bash
   docker-compose up -d
   ```
5. Access the web dashboard at `http://localhost:8083`.
6. If Telegram is configured, send `/start` to your bot to begin receiving notifications.

## Running Locally (without Docker)

1. Install Python 3.11+ and Redis.
2. Create a virtual environment and install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Copy `.env.example` to `.env` and configure.
4. Start Redis (e.g., `redis-server`).
5. Run the bot:
   ```bash
   python -m src.main
   ```

## Telegram Commands

| Command | Description |
|---------|-------------|
| `/start` | Register chat for notifications |
| `/pause` | Pause trading |
| `/resume` | Resume trading |
| `/status` | Show current symbols, positions, and balances |
| `/trades` | Show last 10 trades |
| `/profit` | Show profit/loss summary |

## Web API Endpoints

| Endpoint | Description |
|----------|-------------|
| `GET /health` | Health check (includes Redis status) |
| `GET /api/status` | Current symbols, positions, balances, paused state |
| `GET /api/trades?limit=20` | Recent trades |
| `GET /api/profit` | Profit/loss summary |
| `GET /api/config` | Current configuration (non‑sensitive) |
| `WS /ws` | Real‑time dashboard data (updates every 2s) |
